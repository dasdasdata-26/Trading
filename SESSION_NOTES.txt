HPM Proposal Generator Web - Session Notes
============================================
Date: March 3, 2026

PROJECT OVERVIEW
----------------
- Type: Flask + Vanilla JS web app for generating professional Word (.docx) proposals
- Location: D:\Dropbox\HPM AI\HPMProposal GeneratorWeb
- Run: python app.py or run.bat -> opens browser at http://localhost:5000
- No database -- JSON-based file storage in hpm_proposal/
- Desktop shortcut created: C:\Users\Owner\Desktop\HPM Proposal Generator.bat

KEY FILES
---------
- app.py -- Flask backend (routes, settings API, doc generation, draft management)
- document_generator.py -- Builds .docx proposals using python-docx
- hpm_proposal/settings.json -- App settings (company, user, footer, rate schedule)
- hpm_proposal/templates/ -- Proposal templates (JSON)
- hpm_proposal/drafts/ -- Saved draft proposals
- hpm_proposal/output/ -- Generated .docx/.pdf files
- templates/index.html -- Main proposal form (Jinja2)
- templates/settings.html -- Settings page (Jinja2 + inline JS)
- static/js/app.js -- Frontend logic
- static/css/style.css -- Styling

DEPENDENCIES
------------
- Flask, python-docx, beautifulsoup4 (see requirements.txt)
- Optional: LibreOffice for PDF export

HOSTING ON ANOTHER COMPUTER
----------------------------
1. Copy the entire project folder to the target machine
2. Install Python 3.7+ (check "Add Python to PATH" during install)
3. Open a terminal and run: pip install -r requirements.txt
4. Run: python app.py (or double-click run.bat on Windows)
5. Optional: Install LibreOffice for PDF export

WHAT WAS DONE
-------------

1. FIX: Line Spacing (Fix 1 from fixes_prompt.md)
   - Problem: generated .docx had 2.0 line spacing
   - Root cause: pf.line_spacing = 1.0 did not reliably set w:lineRule in OOXML
   - Fix: rewrote _para_spacing() in document_generator.py to use direct XML
     manipulation, explicitly setting w:before=0, w:after=0, w:line=240,
     w:lineRule=auto on every paragraph
   - Also applied WD_LINE_SPACING.SINGLE to the Normal style so all paragraphs
     inherit single spacing by default
   - File changed: document_generator.py

2. FIX: Exact Indentation (Fix 2 from fixes_prompt.md)
   - Verified indentation values already matched spec exactly:
     Roman numeral headers: w:left=720, w:hanging=720 (label at 0", text at 0.5")
     Lettered items (A/B/C): w:left=1440, w:hanging=720 (label at 0.5", text at 1.0")
     Numbered items (1/2/3): w:left=2160, w:hanging=720 (label at 1.0", text at 1.5")
   - No code changes needed -- already correct

3. FIX: Section Delete / Restore (Fix 3 from fixes_prompt.md)
   - Already implemented from prior session -- verified working
   - Each section panel (I-IV) has an X Delete button in the UI
   - Deleted sections hidden in UI and excluded from .docx output
   - Restore bar appears below sections when any are deleted
   - State persists through Save/Load Draft and New Proposal
   - Only active (non-deleted) sections appear in fee table and TOTAL
   - Files: templates/index.html, static/js/app.js, document_generator.py,
            static/css/style.css

4. FIX: Word Auto-Numbering (Fix 4 from fixes_prompt.md)
   - Already implemented from prior session -- verified working
   - OOXML abstractNum/num definitions written so Word continues A/B/C and
     1)/2)/3) sequences when user adds items after export
   - Each section gets its own concrete numId (resets to A independently)
   - File: document_generator.py (_setup_numbering, _alloc_letter_num,
           _alloc_number_num, _add_list_para)

5. CHANGE: Sub-sub-item format changed from "1." to "1)"
   - In document_generator.py: lvl_text changed from "%1." to "%1)"
   - In static/js/app.js: three places where UI label showed "N." changed to "N)"
   - Files changed: document_generator.py, static/js/app.js

6. CHANGE: Rate schedule special charges now use round bullet points
   - Previously used style="List Bullet" which could conflict with custom
     numbering XML
   - Now uses explicit bullet character (u2022) with hanging indent
   - File changed: document_generator.py (_build_rate_schedule)
